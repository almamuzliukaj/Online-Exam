# 16. Notion Workspace Guide

## Main structure
Create one parent page named `Online Exam System`.

Add these subpages:
- `Project Home`
- `Sprint Board`
- `Product Requirements`
- `Architecture and Data Model`
- `API Contracts`
- `QA and Test Cases`
- `Meeting Notes`
- `Team Tasks`
- `Risks and Decisions`

## Sprint Board database
Use these properties:
- Task
- Owner
- Sprint
- Status
- Priority
- Feature Area
- Due Date
- Blocked By
- Demo Ready

## Professional setup rules
1. Use one shared task database instead of separate notes.
2. Add one filtered view per teammate.
3. Add one filtered view per sprint.
4. Every task template should include goal, acceptance criteria, test cases, and owner.
5. Link screenshots, API examples, and files inside each task card.

## Suggested databases
- `Tasks`
- `Features`
- `Bugs`
- `Decisions`
- `Test Cases`
